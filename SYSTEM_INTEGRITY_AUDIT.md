## D. أخطاء API

### ❌ أخطاء 500 (خطأ واحد)

| المسار | الخطأ | السبب | الخطورة |
|--------|-------|-------|---------|
| `GET /api/well-expenses/list` | `500` | `well_id = NaN` — المسار يتوقع `?wellId=<number>` لكن لا يتحقق من القيمة قبل إرسالها للـ SQL. عند الاستدعاء بدون `wellId` أو بقيمة غير رقمية يحدث `invalid input syntax for type integer: "NaN"` | **Medium** — يعمل بشكل صحيح عند تمرير `wellId` صالح |

**الملف المسبب:** `server/routes/modules/wellExpenseRoutes.ts:55-65`
**الحل:** إضافة تحقق من `wellId` قبل الاستعلام.
