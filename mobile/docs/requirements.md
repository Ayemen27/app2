## 3. سيناريوهات الاستخدام (Use Cases)

### UC-01: تسجيل حضور عامل (Offline)
```
الفاعل: مهندس ميداني
المتطلبات: الهاتف بدون إنترنت في موقع البناء

1. يفتح تطبيق إدارة المشاريع
2. يختار المشروع الحالي
3. يضغط "تسجيل حضور"
4. يختار العامل من القائمة
5. يدخل عدد أيام العمل والمبلغ
6. يضغط "حفظ"
7. يظهر مؤشر "محفوظ محلياً - ينتظر المزامنة"

النتيجة: البيانات محفوظة في SQLite، مضافة لطابور المزامنة
```

### UC-02: مزامنة عند عودة الإنترنت
```
الفاعل: النظام (تلقائي)
الشرط: توفر اتصال إنترنت

1. يكتشف النظام توفر الإنترنت
2. يقرأ طابور المزامنة
3. يرسل البيانات للسيرفر POST /api/sync/push
4. يستقبل التحديثات GET /api/sync/changes
5. يحل أي تضارب
6. يحدث قاعدة البيانات المحلية
7. يعرض "تمت المزامنة بنجاح"

النتيجة: جميع البيانات متزامنة
```

### UC-03: تصوير فاتورة مواد
```
الفاعل: مهندس مشتريات
المتطلبات: كاميرا الهاتف

1. يفتح "شراء مواد"
2. يدخل بيانات الفاتورة
3. يضغط "إرفاق صورة"
4. تفتح الكاميرا
5. يلتقط صورة الفاتورة
6. يؤكد الصورة
7. يتم ضغط الصورة وحفظها محلياً
8. يضغط "حفظ"

النتيجة: الفاتورة محفوظة مع الصورة، جاهزة للمزامنة
```
