## 🤔 أسئلة عامة

### س: أين أبدأ إذا كنت وكيل جديد؟
**ج**: 
1. اقرأ `replit.md` (هذا الملف الرئيسي)

**ج**: أضفه هنا في هذا الملف تحت قسم جديد:

```markdown
**التاريخ**: 2025-12-24
```

### س: هل يمكنني تعديل package.json؟
**ج**: لا، إلا بإذن من المشروف. الأدوات مثبتة بالفعل.

## 💻 أسئلة تقنية

### س: ماذا يعني TypeScript Check؟
**ج**: 
```bash
npm run check
```
هذا يتحقق من أن الكود TypeScript سليم (بدون أخطاء types).

### س: إذا كان هناك TypeScript error ماذا أفعل؟
**ج**:
1. اقرأ الخطأ بعناية
2. اذهب إلى السطر المشار إليه
3. صحّح الخطأ (عادة ما يكون type mismatch)
4. أعد تشغيل `npm run check`

### س: كيف أختبر Service Worker؟
**ج**:
1. شغّل `npm run dev`
2. افتح DevTools: `F12`
3. اذهب إلى tab `Application`
4. اختر `Service Workers` من اليسار
5. يجب أن ترى الـ SW مع status "activated and running"

### س: كيف أختبر Offline؟
**ج**:
1. افتح DevTools: `F12`
2. اذهب إلى tab `Network`
3. ستجد checkbox يسمى `Offline` - اختره
4. الآن التطبيق بدون إنترنت
5. أعد تحميل الصفحة، يجب أن تحمل من الـ Cache

## 🚀 أسئلة عن المراحل

### س: هل يجب أن أنهي كل المرحلة قبل الانتقال للتالية؟
**ج**: نعم، المراحل مترابطة (dependencies). لا تنتقل للمرحلة 2 حتى تنتهي المرحلة 1 بالكامل.

### س: ماذا لو واجهت مشكلة في وسط المرحلة؟
**ج**: 
1. وثّقها في `BLOCKERS.md`
2. حاول إيجاد حل بديل
3. إذا استمرت المشكلة، أخبر المشروف
4. لا تتحرك بدون حل

## 🧪 أسئلة عن الاختبار

### س: كيف أعرف أن مهمتي نجحت؟
**ج**: راجع `ACCEPTANCE_CRITERIA.md` للمهمة الخاصة بك. إذا اجتازت جميع المعايير = نجح.

### س: هل يجب أن أختبر الأشياء يدويًا؟
**ج**: نعم، اختبر يدويًا أولاً. لا يوجد automated tests في الوقت الحالي.

### س: ماذا أفعل إذا كان هناك عدة طرق لتنفيذ شيء؟
**ج**: اختر الطريقة التي تتبع معايير القبول وأكثر وضوحًا وسهولة.

## 🎯 أسئلة عن الأهداف

### س: ما هو الفرق بين PWA و Android App؟
**ج**:
- **PWA**: تطبيق ويب بميزات جديدة (offline, push، قابل للتثبيت)
- **Android App**: تطبيق أندرويد يستخدم نفس الكود PWA عبر Capacitor
- **الميزة**: كود واحد يعمل على الويب والـ Android معاً

### س: هل يجب أن أهتم بـ iOS أيضاً؟
**ج**: لا، خارج نطاق هذا المشروع. iOS يمكن إضافتها لاحقًا.

### س: متى ننتهي من كل شيء؟
**ج**: حسب الجدول الزمني:
- الأسبوع 1: PWA + IndexedDB
- الأسبوع 2: Sync + Firebase
- الأسبوع 3: Capacitor
- الأسبوع 4: Testing + APK

## 📝 أسئلة عن التوثيق

### س: كيف أكتب commit message؟
**ج**:
```bash
git commit -m "Phase X: task description (مختصر)"
```
مثال:
```bash
git commit -m "Phase 1.1: Add manifest.json for PWA"
git commit -m "Phase 2.1: Setup IndexedDB database"
```

### س: أين أكتب الملاحظات أثناء العمل؟
**ج**: في الملفات التالية:
- `NOTES.md`: ملاحظات عامة
- `BLOCKERS.md`: مشاكل معلقة
- `QUESTIONS.md`: أسئلة (هذا الملف)

## 🤝 أسئلة عن التعاون

### س: كيف أخبر الوكيل التالي أنني انتهيت؟
**ج**: 
1. أكتب commit نظيف
2. وثّق ملاحظاتك في `NOTES.md`
3. أضف أي مشاكل في `BLOCKERS.md`
4. أبلّغ المشروف أنك انتهيت

### س: ماذا لو وجدت خطأ من الوكيل السابق؟
**ج**:
1. وثّقه في `BLOCKERS.md`
2. حاول إصلاحه إذا كان بسيط
3. إذا كان معقد، أخبر المشروف

## 🆘 أسئلة في حالات الطوارئ

### س: ماذا لو كسرت التطبيق كاملاً؟
**ج**: 
1. لا تقلق، الـ git و checkpoints موجودة
2. اطلب rollback إلى checkpoint سابق
3. ابدأ من جديد

### س: ماذا لو لم أستطع فهم المهمة؟
**ج**:
1. اقرأ ACCEPTANCE_CRITERIA مرة أخرى بتأني
2. اسأل في QUESTIONS.md
3. اطلب شرح من المشروف
