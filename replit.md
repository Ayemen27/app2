# مشروع AXION (أكسيون) - إدارة العمليات المتكامل

## نظرة عامة
نظام متطور لإدارة عمليات الآبار، التقارير المالية، والمراقبة اللحظية، مدعوم بوكيل ذكاء اصطناعي داخلي. الهوية البصرية تعتمد على الدمج بين العربية والإنجليزية (AXION | أكسيون).

## بروتوكول العمل (قواعد صارمة)
- **الدقة فوق السرعة:** لا يتم تقديم حلول غير مؤكدة.
- **التحليل المنطقي:** التفكير خطوة بخطوة قبل كل إجابة.
- **الواقعية:** اختيار الحلول القابلة للتنفيذ وتقييم التكلفة/السرعة.
- **الشفافية:** التصريح الصريح عند نقص المعلومات أو عدم المعرفة.
- **منع المجاملة:** تقديم الواقع التقني كما هو.
- **التفكير خارج الصندوق:** اقتراح حلول بديلة ذكية عند الحاجة دون تهور.

## الهيكل التقني
- **Backend:** Express.js مع Drizzle ORM (PostgreSQL).
- **Frontend:** React + Tailwind CSS + Shadcn UI.
- **Data Logic:** Offline-first باستخدام IndexedDB ومزامنة WebSocket.
- **Layout:** هيكلية LayoutShell مع دعم كامل للـ Sidebar (RTL).
- **Observability:** SigNoz لتجميع البيانات و Sentry لتتبع الأخطاء (SDK Integrated).
- **Services:** إدارة أسرار، ونظام مزامنة متقدم (isLocal, synced, pendingSync).

## بروتوكول الذكاء الاصطناعي (AI Agent Rules)
1. **التحليل قبل الإجابة:** التفكير خطوة بخطوة ملزم.
2. **الصدق التقني:** لا اختلاق، لا مراوغة، ولا تعميم.
3. **الأولوية:** الحل الأكثر واقعية وقابلية للتنفيذ هو المفضل.
4. **نسبة اليقين:** توضيح الشكوك والافتراضات قبل البناء عليها.
5. **المهنية:** الحفاظ على نبرة محايدة ومهنية حتى في حالات الفشل.
