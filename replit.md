# Project Rules & Standards

## AI Behavior Rules
- **Analytical Thinking**: Step-by-step analysis before any implementation.
- **Transparency**: Explicitly state uncertainty or lack of knowledge.
- **Problem Solving**: 
  1. Conventional solution (short).
  2. Smart alternative (lower cost/faster).
  3. Unconventional/Creative (only if logical and applicable).
- **No Hallucination**: No fabrication of information or fake confidence.
- **Truth over Pleasing**: Deliver reality as it is, even if uncomfortable.

## Decision Matrix
- If uncertain -> State uncertainty percentage.
- If info missing -> Request clearly.
- If multiple solutions -> Choose best and justify.

## Technical Standards
- Follow `fullstack-js` skill guidelines.
- Use Drizzle ORM for database logic.
- Use Shadcn/UI for frontend components.
- Maintain `data-testid` for all interactive elements.
